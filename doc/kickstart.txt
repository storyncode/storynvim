================================================================================
INTRODUCTION                                                  *kickstart.nvim*

`kickstart.nvim` remains the stable help entrypoint for this fork, even though
the runtime is now organized as a modular Neovim configuration instead of a
single main config file.

                                                            *kickstart-is-not*
It is not:
- A complete framework for every plugin under the sun
- A monolithic config where `init.lua` is the main place to customize everything

                                                                *kickstart-is*
It is:
- A practical personal or forkable configuration with the most common "IDE"
  style features already wired up:
    - autocompletion
    - goto-definition
    - find references
    - fuzzy finding
    - and room to extend it further
- A modular layout that is easier to navigate and change safely:
    - `init.lua` only sets early globals and calls `require('story.bootstrap').setup()`
    - `lua/core/` owns always-on editor behavior such as options, keymaps,
      diagnostics, and autocmds
    - `lua/plugins/` owns the shipped plugin specs and configuration
    - `lua/custom/plugins/` is the stable fork-local path for your own plugins
      and overrides
- A place to make the config your own without guessing where code lives.
    - Browse the module that matches the change you want instead of treating
      `init.lua` as the main customization surface.
    - `lua/kickstart/plugins/` remains available as a compatibility/example
      namespace, not the preferred path for new customization in this fork.
