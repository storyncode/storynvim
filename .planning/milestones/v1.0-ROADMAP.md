# Milestone v1.0: StoryNvim Modular Migration

**Status:** SHIPPED 2026-04-12
**Phases:** 1-5
**Total Plans:** 14

## Overview

This milestone migrated the repo from a monolithic Kickstart-style `init.lua` to a modular Neovim configuration with a thin entrypoint, explicit startup orchestration, concern-based plugin imports, extracted always-on editor modules, compatibility-preserving extension paths, and updated documentation that matches the shipped architecture.

## Phases

### Phase 1: Bootstrap Skeleton

**Goal**: Create the new startup structure so `init.lua` becomes a thin bootstrap layer that delegates to modules without breaking startup behavior.
**Depends on**: Nothing (first phase)
**Plans**: 3 plans

Plans:

- [x] `01-01-PLAN.md` — Define the target startup/module layout and move the ordered startup body into a bootstrap module
- [x] `01-02-PLAN.md` — Reduce `init.lua` to the minimal entrypoint and verify the real startup path still boots cleanly
- [x] `01-03-PLAN.md` — Extract the risky `lazy.nvim` bootstrap into a shared helper and rerun the smoke suite

**Details:**
- Goal achieved with a thin `init.lua`, ordered bootstrap delegation, and extracted `lazy.nvim` helper.

## Milestone Summary

**Key Accomplishments:**

- Replaced the monolithic startup with a thin `init.lua`, ordered bootstrap module, and dedicated lazy bootstrap helper.
- Moved shipped plugin specs into concern-based `lua/plugins` modules and wired bootstrap through `lazy.nvim` imports.
- Extracted always-on options, diagnostics, keymaps, and autocmds into `lua/core/*` with reusable smoke coverage.
- Preserved compatibility by activating `custom.plugins`, keeping legacy Kickstart overlays working, and adding parity assertions.
- Updated README, help docs, and inline comments to match the modular runtime now shipped.

**Key Decisions:**

- Treated the migration as a brownfield structural refactor rather than a rewrite.
- Targeted a minimal `init.lua` plus explicit module boundaries.
- Used `lua/plugins` concern modules as the canonical shipped plugin source.
- Kept `lua/custom/plugins/` as the stable fork-local extension path and `lua/kickstart/plugins/` as compatibility/example-only.

**Issues Resolved:**

- Fixed repo-root `runtimepath` behavior so headless `nvim -u ./init.lua` verification works through the real entrypoint.
- Removed the shipped inline plugin table without breaking `lazy.nvim` health or Kickstart health checks.
- Recovered reusable verification coverage for extracted Phase 3 core-module behavior.

**Issues Deferred:**

- Phase 01 validation remains draft and not Nyquist-compliant.
- Phase 02 validation remains draft and not Nyquist-compliant.

**Technical Debt Incurred:**

- `README.md` still contains one stale upstream phrase: `Install Kickstart`.
- `lua/kickstart/plugins/gitsigns.lua` still contains a historical comment that references `init.lua` rather than the shipped `lua/plugins/editor.lua` source.
